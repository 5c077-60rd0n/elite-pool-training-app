# Fargo Climb - Elite Pool Training PWA

Offline-first, elite-focused training system for players who want a daily execution loop and long-horizon Fargo growth.

This app is built around one core idea: every session should create signal, and every signal should improve the next session.

## What This App Does

Fargo Climb combines 4 layers in one workflow:

1. Structured daily logging.
2. Adaptive planning and recovery recommendations.
3. Match-performance tooling (prep cards, simulations, post-event analysis).
4. Elite-performance modules for decision quality, pressure, readiness, and pattern work.

## Feature Map (What + Why + How)

### 1) Onboarding

What: Sets player profile and baseline training context.

Why: Personalized guidance (rating targets, phase, weekly progression) depends on profile values.

How:

1. Complete onboarding once at first launch.
2. You are automatically routed to main app screens after onboarding is complete.

### 2) Dashboard

What: Central command center for rating progress, KPI trends, confidence index, gamification, season ladder, notification intelligence, and readiness snapshots.

Why: You need one place to decide what to do next based on performance, not emotion.

How:

1. Check Rating Progress card for current/estimated Fargo and phase.
2. Review Adaptive Next Session Plan and Recovery Protocol cards.
3. Use Confidence, Personal Records, and Match Readiness to pick focus.
4. Watch Fargo and KPI charts for trend direction, not single-day noise.

### 3) Daily Session Flow (Today Session)

What: Required daily log with workbook-style fields (shotmaking, bullseye, ghost, safety, line-up, lesson completion, notes).

Why: This is the engine that drives weekly summaries, adaptive planning, confidence modeling, seasonal progression, and records.

How:

1. Read the daily template and adaptive card at top.
2. Run your practice.
3. Fill in session fields and save.
4. Repeat daily for clean trend data.

### 4) Adaptive Daily Plan

What: KPI-focused next-session targets and checklist generated from recent performance.

Why: It prevents random practice by selecting the highest leverage KPI for the next session.

How:

1. Use the focus KPI, rationale, and target metrics as your daily script.
2. Follow checklist items in order.
3. Match session length to recommendation for load control.

### 5) Recovery Recommendation Plan (3-Day)

What: Short horizon recovery protocol triggered by weak sessions and/or competition signals.

Why: Stops performance dips from compounding.

How:

1. Use severity, trigger, and recommended focus area to reshape the next 3 days.
2. Prioritize listed actions before adding new work.

### 6) Weekly Schedule

What: Structured weekly layout for session themes and drill emphasis.

Why: Long-term gains require cadence, not isolated hard days.

How:

1. Use it for planning your week before sessions start.
2. Keep Daily Session logs aligned with the schedule intent.

### 7) Progress

What: Historical review surface for long-term training and rating evolution.

Why: Progress decisions should be made on trend lines and consistency, not memory.

How:

1. Review historical KPI behavior and compare blocks.
2. Identify lagging metrics to feed into your next adaptive cycle.

### 8) KPI Tracker

What: KPI-specific tracking against benchmark ranges and targets.

Why: Lets you see which systems are moving and which are stalled.

How:

1. Focus on one lagging KPI at a time.
2. Pair KPI findings with Today Session and Elite Lab bridge actions.

### 9) Phase Overview

What: Fargo-phase framework for progression milestones.

Why: Provides context for what "good progress" means at your current level.

How:

1. Use phase expectations to set realistic short-term goals.
2. Align milestones and practice blocks with your current phase.

### 10) Milestone Log

What: Verification workflow for milestone attempts with notes and optional clip/snapshot references.

Why: Distinguishes real validated progress from subjective impressions.

How:

1. Log attempt outcome (pass/fail).
2. Include evidence notes and optional clip/snapshot context.
3. Re-test until milestone status flips to met.

### 11) Mechanics Audit

What: Weekly technique checkpoints and audit logging.

Why: Protects stroke fundamentals while you push performance intensity.

How:

1. Run checklist items weekly.
2. Mark pass/needs work.
3. Add targeted action item to next sessions when a checkpoint drifts.

### 12) Mental Game

What: Mental-performance practice surface.

Why: Execution under pressure is partly psychological and must be trained explicitly.

How:

1. Use before hard sessions/matches.
2. Capture reflections and keep protocols repeatable.

### 13) Tournament Prep

What: 14-day prep planner with checklist progression, opponent card attachments, and post-event analysis.

Why: Converts tournament prep from vague intention to a repeatable system.

How:

1. Create event plan (name/date/format/location).
2. Work checklist each day to completion.
3. Attach/update opponent prep card.
4. Save post-event analysis to feed recovery focus.

### 14) Competition Log

What: Structured event outcome history.

Why: Competition outcomes influence confidence, recovery logic, and planning quality.

How:

1. Log each event result and notes.
2. Include key error themes and follow-up priorities.

### 15) Match Simulator

What: Race-format simulation with pressure metrics and readiness scores.

Why: Practice readiness and match readiness can diverge; this quantifies the gap.

How:

1. Run simulation sessions before events.
2. Log pressure attempts, hill-hill result, and outcome.
3. Compare match readiness vs drill readiness and close the gap.

### 16) Opponent Prep Cards

What: Reusable opponent archetype cards with opening patterns, safety plans, and bailout choices.

Why: Reduces in-match cognitive load and improves first-inning decisions.

How:

1. Create or update card in Tournament Prep.
2. Keep entries concise and specific.
3. Attach card to active tournament plan.

### 17) Confidence Index + Personal Records

What: Performance insights layer that computes confidence signals and tracks personal bests.

Why: Confidence should be evidence-based, and PRs show hard proof of improvement.

How:

1. Use confidence components to identify which area is suppressing total confidence.
2. Use PRs to benchmark training quality across blocks.

### 18) Gamification + Seasons

What: XP, levels, streaks, weekly quests, themed quest chains, and boss challenges.

Why: Increases adherence and adds progression pressure without replacing real KPIs.

How:

1. Treat quests as consistency constraints.
2. Use season chain/boss progress as accountability signals.
3. Do not chase XP at the expense of KPI quality.

### 19) Elite Performance Lab (All 10 Elite Modules)

What: Advanced high-performance workspace with:

1. Shot Decision IQ Engine.
2. Pressure Scenario Generator.
3. Opponent Pattern Intelligence.
4. Break Optimization Lab.
5. Safety/Kick Systems Trainer.
6. Rack Pattern Quality Scoring.
7. Pre-Shot Routine Compliance.
8. Tournament Debrief Autopsy.
9. Elite Benchmark Mode.
10. Fatigue + Readiness Layer.

Why: Helps high-level players train decision quality, pressure execution, and physical/mental readiness as first-class systems.

How:

1. Log entries inside each module (not just outcomes).
2. Use benchmark deltas to set improvement targets.
3. Use readiness score to adjust training intensity before sessions.

### 20) Elite -> Today Session Bridge

What: One-tap actions in Elite Lab that push elite priorities into the Adaptive Daily Plan checklist.

Why: Prevents elite insights from staying isolated in a separate screen.

How:

1. In Elite Lab, open Elite -> Today Session Bridge.
2. Tap one of:
	- Promote Latest Tournament Priority.
	- Promote Weakest System Focus.
	- Promote Readiness-Based Plan.
3. Open Daily Session Flow and execute promoted checklist item(s).

### 21) Settings, Backup, Import, Reset, and Coach Export

What: Full data operations and configuration center.

Why: Protects your training history and enables coach review workflows.

How:

1. Update profile and rating fields.
2. Configure reminders/notifications and gamification toggles.
3. Export full backup JSON regularly.
4. Import backup JSON on new devices or recovery.
5. Export coach review payload JSON for external review.
6. Reset all local data only when intentionally restarting.

All major tracker domains are included in backup/import/reset, including Elite Lab state.

### 22) Notifications + Sync Queue

What: Local reminder notifications and smart alert intelligence, with pending log sync metadata.

Why: Keeps training cadence and protects data flow while working offline.

How:

1. Grant notification permission.
2. Set reminder time in Settings.
3. Smart alerts trigger based on recent training/recovery context.
4. Pending logs clear automatically when device returns online.

## Recommended Daily Operating Flow

1. Open Dashboard and read Adaptive + Recovery cards.
2. If needed, open Elite Lab and promote one elite priority.
3. Run session and log in Daily Session Flow.
4. Review Confidence + Readiness + KPI deltas.
5. Before events, run Match Simulator and Tournament Prep.
6. After events, log Competition + Debrief, then feed next week.

## Offline-First Behavior

- Uses `vite-plugin-pwa` + Workbox service worker
- Uses an inject-manifest service worker with explicit cache policies for pages, static assets, images, fonts, and local JSON feeds
- Static assets are precached after first online load and outdated caches are cleaned automatically
- Supports update prompts, background update checks, and one-tap reload into the newest version
- Keeps core coaching workflows available offline with automatic sync retries when the app returns online

## Data, Storage, and Safety Notes

- App data is stored locally (IndexedDB via `idb-keyval`)
- Backups are JSON files exported from Settings
- Import supports current and legacy payload shapes
- Use regular exports if your device is mission-critical for training data
